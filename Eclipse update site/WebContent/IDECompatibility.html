Keeping the eclipse project compatible with the IDE

Currently the arduino eclipse plugin is not compatible with the Arduino IDE. The incompatibility is caused by the fact that the arduino eclipse pligin works with C++ files only. The arduino IDE can handle C++ files but needs at least one PDE file (for amongst others) to identify the classpath.
To be able to compile your code directly in the Arduino IDE you need to do something extra at three common actions doring Arduino code development
1) After installation tell Eclipse to process ino files like C++ file
2) After creation of a new sketch project tell the compiler to treat pde files like C++ files (-x c++ compiler option) and rename cpp file to ino (or pde if you are using a version below 1.0)
3) After inclusion of a library add an include of the library to the ino file.
4) In the Arduino IDE preferences set the flag "use external editor" on
5)in the arduino IDE open the ino file from step 2.

Note that if an imported library contains examples you will have to move them to another location on your disk in the current version of the plugin.

And here are detailed instructions on how to do this

After installation tell Eclipse to process ino like C++ file

In the menu select Windows ->preferences
In the section C/C++ -> file types press new
enter *.pde as filename and C++Source file as type.

After creation of a new sketch project tell the compiler to treat ino files like C++ files

right click on the project and select properties
In the section C/C++ -> settings select the tab tool settings (normally already selected)
In the section AVR C++ compiler change the command line pattern by adding -x c++ before ${INPUTS}
select OK to save the settings.

After creation of a new sketch rename cpp file to pde

Right click the .cpp file created by the new arduino sketch wizard
select rename
change the .cpp to .ino

After inclusion of a library add an include reference to the library to the ino file.

In the ino file at the top add a line that looks like
#include < [the imported library].h >
